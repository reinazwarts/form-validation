JavaScript Form validation, no plugins, no jQuery

Why to use

  • Uses existing browser validation API
  • No need to create error messages, they are there already (in the users browser language)
  • Itâ€™s minified size is 3kb (jQuery validation plugin is 23kb)
  • Implementation is very easy

How to use

  1. Add class

    - Give your HTML form element the css class 'form-validate' to apply, like this: <form class="form-validate">

  2. Add required

    - Give the required HTML fields element the required attribute to make it... required :)
    <input type="text" required id="myinput" />.

  3. Add JavaScript

    - Include the javascript file from the dist folder at the bottom of your HTML, before the closing body tag, like this: <script src="form-validation.min.js"></script>. Or include the javascript file from the src folder to your preprocessor package. This src file is ES2015, so you will need Babel or another converter.

  4. Add CSS

    - Grap from the sccs files in the src/scss folder what you need. Include at least this CSS to your project (make it more specific if needed): 
    .is-hidden {
    display: none;
    visibility: hidden;
}
Be sure to add the first option with an empty value to make the validation work, like this: <option value="">please select</option>


Error messages

By default the error messages come from the browser itself (in the language set by the user) and do not need to be specified. What is really cool and handy, because you don't need to manage this content at all, awesome! If you need custom messages from a CMS, read futher for instructions.



Custom error messages from a CMS (optional)

Add a data attribute to the field element, data-validation-message-* where the * must correspond with the correct validaty value. Most used will probably be 'valuemissing' and 'typemismatch' (e-mailaddresses), Example: <input type="text" data-validation-message-valuemissing="Your custom message for value missing" id="myinput" />



Custom error messages static from JavaScript (not recommended)

Another option is static error messages defined in JavaScript by your team/client. I would't do this, only if the other two options are absolutely not possible. In that case, edit the form-validation.js file. Uncomment the rules in the errorMessages object and edit the values as you wish. Or you can place this following JavaScript code somewhere (in another JavaScript file). Place here your static custom texts for error messages.

formValidation.errorMessages = {
  badInput: 'type error message here',
  customError: 'type error message here',
  patternMismatch: 'type error message here',
  rangeOverflow: 'type error message here',
  rangeUnderflow: 'type error message here',
  stepMismatch: 'type error message here',
  tooLong: 'type error message here',
  tooShort: 'type error message here',
  typeMismatch: 'type error message here',
  valueMissing: 'type error message here'
}
